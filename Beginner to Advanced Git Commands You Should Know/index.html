Beginner to Advanced Git Commands You Should Know

June 23, 2025 Git 9 min read

As a Computer Science student, learning Git was a game-changer for me. At first, it felt confusing—commits, branches, merges—but over time, Git became my best friend for managing code. Whether you're working on solo projects or collaborating with a team, knowing Git commands is essential. In this blog, I'll take you from beginner to advanced Git commands so you can confidently manage your projects.

Why Git is Important

Git is a version control system that tracks changes in your code. It allows you to:

  • Save multiple versions of your code
  • Collaborate with teammates without overwriting each other's work
  • Revert changes when things go wrong
  • Maintain a clean, organized code history

Learning Git commands is like learning the vocabulary of software development—it's essential for any coder.

Beginner Git Commands

These are the commands you'll use most often when starting with Git:

1. git init

Initializes a new Git repository in your project folder.

git init

2. git clone

Copies an existing repository to your local machine.

git clone https://github.com/username/repository.git

3. git status

Shows the current status of files—tracked, untracked, or modified.

git status

4. git add

Stages changes to be committed.

git add filename    # Add a single file
git add .           # Add all changes

5. git commit

Saves your staged changes with a message.

git commit -m "Initial commit"

6. git push

Uploads your changes to a remote repository.

git push origin main

7. git pull

Downloads the latest changes from a remote repository.

git pull origin main

Intermediate Git Commands

Once you're comfortable with the basics, these commands help you work with branches and history:

1. git branch

Shows existing branches or creates a new branch.

git branch           # List branches
git branch feature   # Create a new branch

2. git checkout

Switches to a different branch.

git checkout feature

Note: Modern Git also supports git switch for changing branches, which is more intuitive.

3. git merge

Combines changes from one branch into another.

git checkout main
git merge feature

4. git log

Shows the commit history.

git log

For a more compact view, use git log --oneline.

5. git diff

Shows changes between commits, branches, or working directory.

git diff

Advanced Git Commands

These commands give you more control over your Git workflow and history:

1. git rebase

Moves or combines a sequence of commits to a new base. Keeps a clean history.

git checkout feature
git rebase main

Warning: Only rebase commits that haven't been pushed to a shared repository.

2. git stash

Temporarily saves changes without committing. Useful when switching branches.

git stash          # Save changes
git stash apply    # Apply saved changes
git stash list     # List all stashes

3. git cherry-pick

Applies a specific commit from another branch.

git cherry-pick <commit-hash>

Useful when you want to bring a specific fix or feature from one branch to another without merging the entire branch.

4. git reset

Undo commits or changes. Be careful—this can modify history.

git reset --soft HEAD~1   # Undo last commit but keep changes
git reset --hard HEAD~1   # Undo last commit completely

Use --soft to keep your changes staged, or --hard to discard them completely.

5. git tag

Mark important points in your project (like releases).

git tag v1.0
git push origin v1.0

Tags are useful for marking releases and can be annotated with messages using git tag -a v1.0 -m "Release version 1.0".

Tips for Using Git Effectively

  • Commit Often: Smaller commits make tracking changes easier and help you identify problems faster.
  • Use Meaningful Messages: Clear commit messages help you and your team understand changes. Write them in present tense, like "Add user authentication" not "Added user authentication".
  • Work on Feature Branches: Keeps the main branch stable and deployable. Create a branch for each feature or bug fix.
  • Pull Before Push: Avoid conflicts by syncing with the remote repository first. Always git pull before git push.
  • Review Your Changes: Use git diff and git status before committing to make sure you're committing what you intend to.
Git is more than a tool—it's a skill every developer should master. From the basics like git init and git commit to advanced commands like git rebase and git cherry-pick, knowing Git helps you work faster, collaborate efficiently, and maintain clean code history.

What Should CS Students Take Away?

  • Start with the basics—master git add, git commit, and git push before moving to advanced commands
  • Practice on personal projects—experiment with branches and merges without fear of breaking anything
  • Learn to read Git history—understanding git log helps you see how projects evolve
  • Don't be afraid of mistakes—Git has safety nets, and you can almost always recover from errors
  • Use Git for everything—even solo projects benefit from version control

Start practicing these commands today, and you'll see how much smoother coding projects can become—whether it's for school assignments, personal projects, or professional development. The more you use Git, the more natural these commands will feel, and soon you'll be managing code like a pro.
